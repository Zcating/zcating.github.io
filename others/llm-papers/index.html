🔥 HuggingFace 热门 LLM 论文 TOP 10

1️⃣ LLM in a flash (260票)

高效内存受限设备的 LLM 推理

🔗 huggingface.co/papers/2312.11514

摘要：解决在有限 DRAM 容量的设备上高效运行 LLM 的挑战。提出两种技术：

• Windowing：通过重用先前激活的神经元来减少数据传输

• Row-Column Bundling：根据 flash 顺序数据访问特点，增加读取的数据块大小

效果：可运行比可用 DRAM 大两倍的模型，CPU 推理速度提升 4-5 倍，GPU 提升 20-25 倍

2️⃣ Executable Code Actions (188票)

用代码执行替代 JSON 生成

🔗 huggingface.co/papers/2402.01030

摘要：提出 CodeAct 范式，用可执行 Python 代码统一 LLM 代理的动作空间。集成 Python 解释器，支持多轮交互中动态修正动作。

效果：在 API-Bank 基准测试中成功率提升高达 20%

3️⃣ LLM Agent Operating System (73票)

LLM 代理操作系统 AIOS

🔗 huggingface.co/papers/2403.16971

摘要：解决 LLM 代理部署中的挑战：调度优化、上下文维护、异构代理集成。提出 AIOS 架构，优化资源分配、支持上下文切换、并发执行、工具服务、访问控制。

4️⃣ Enhancing Human-Like Responses (61票)

让 LLM 更像人

🔗 huggingface.co/papers/2501.05032

摘要：探索让大语言模型更拟人化的技术，包括：用多样化数据集微调、融入心理学原理、设计更模仿人类推理模式的模型。

意义：提升用户体验，拓展 AI 应用边界

5️⃣ CALM - LLM Augmented LLMs (38票)

模型组合扩展能力

🔗 huggingface.co/papers/2401.02412

摘要：提出 CALM 方法，通过跨注意力机制组合不同模型。保留原有模型权重，新增少量参数即可扩展能力。

效果：PaLM2-S 搭配小模型，低资源语言翻译提升 13%，代码生成相对提升 40%

6️⃣ Judging the Judges (37票)

LLM 作为评判者的评估与偏见

🔗 huggingface.co/papers/2406.12624

摘要：全面研究 LLM 作为评判者的表现。使用 TriviaQA 评估 9 个评判模型。

发现：Llama-3 70B 和 GPT-4 Turbo 与人类对齐良好；强调使用 Cohen's kappa 的重要性

7️⃣ LLM-Rec (28票)

LLM 做个性化推荐

🔗 huggingface.co/papers/2307.15780

摘要：研究通过提示工程增强 LLM 个性化推荐性能。提出四种提示策略：基础提示、推荐驱动提示、参与度引导提示、组合提示。

效果：结合原始内容描述和 LLM 增强输入，提升推荐效果

8️⃣ Self-Extend (27票)

无需微调扩展上下文窗口

🔗 huggingface.co/papers/2401.01325

摘要：提出 Self-Extend 方法，激发 LLM 本身处理长上下文的潜力。构建双层注意力信息：组级和邻居级。

效果：仅需修改 4 行代码，无需微调即可扩展现有 LLM 上下文窗口

9️⃣ VoiceAssistant-Eval (23票)

语音 AI 助手 benchmark

🔗 huggingface.co/papers/2509.22651

摘要：全面评估 AI 助手在听、说、看方面的能力。包含 10,497 个样本，13 个任务类别。

发现：开源模型不一定输闭源模型；大多数模型擅长"说"但听力理解较弱

🔟 Hydragen (20票)

高吞吐量推理优化

🔗 huggingface.co/papers/2402.05099

摘要：针对共享前缀的批量推理优化。分解共享前缀和唯一后缀的注意力计算。

效果：端到端吞吐量提升高达 32 倍；支持超长共享上下文
